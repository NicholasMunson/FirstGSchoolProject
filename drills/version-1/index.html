  • 1:Set/Clear the stage

    For me, this is done by first by putting myself in an environment that is best for learning. I am a very social person so I need to be away from social situations. I find a nice quiet place with very little distractions. I thin turn off all my notifications on my cell and mac.     2: Clarify what and why.

    What:“Today I am going to learn basic commands in the command line/ oh my zsh, and how it works.” Why: “ I am learning this because command line is a faster and more powerful tool then using the gooies. Command Line when used with oh my zsh can be unstoppable”.

  • Play

  • 1: wrestle with the task.

    Think about different ways something can be done. Try something that can be faster and more efficient. So in Command Line, try to use the command “touch” to create multiple files at the same time “ touch index.html main.css”. If something more complex is required wrestle with it. Try many different ways to make something work. The worst thing that happens is that it does not work.

    2: Take Risks!

    In command line it is kind of hard to take risks, but getting out of the comfort zone is what this is all about. Trying something new is a way to grow. Can I make a directory and a file at the same time with the file staying in that directory? If so how would I do that? maybe this “mkdir test touch ./test/newfolder”… No it just made directories “test”, “touch”, and “newfloder”, but “newfolder” was put into the “test” directory. I cannot make touch files and mkdir directories at the same time but I can make new directories at the same time and nest them at the same time. I learned something I did not set out to learn so, that is awesome! I found that can remove all of these directories at the same time without a force! Pretty cool stuff!

    3: Have Fun!

    Working with new technologies is always exciting. The best way to really figure them out is by playing with them. New technologies are sometimes scary because they are new. It is best to keep a level head that is focused on growth and not a fixed mindset that does not allow growth. When a situation pops up that is challenging do not say “I can’t do this” or “this is to hard” instead think about it in a different approach. Think about these challenges as opportunities to grow. Try many things that may or may not work. Having fun is something that can benefit you later. In command line I tried to making two directories at the same time like I can with the “touch” command. It worked! Now I do not need those directories so lets remove them. can I remove more then one at a time? Yes!
